## 🎨 设计理念

### 为什么流程比按钮更重要？

#### 1. **完整性 > 单点操作**
```
❌ 只看到按钮：
   "我该点这个按钮吗？"
   "点了会发生什么？"
   "我准备好了吗？"

✅ 看到完整流程：
   "哦，原来要5步"
   "我现在在第3步"
   "下一步点启动就行了"
```

#### 2. **理解 > 执行**
```
用户需要先理解系统如何工作，
然后才有信心执行操作。

流程图 = 系统的"说明书"
```

#### 3. **掌控感 > 效率**
```
一个巨大的按钮可能很高效，
但用户会缺乏掌控感。

看到完整流程，用户知道：
- 我在哪里
- 还要做什么
- 预期结果是什么
```

## 🎯 设计哲学

### 以用户为中心的3个层次

#### Level 1：我在哪里？（定位）
```
→ 投递流程显示"当前第3步"
→ 用户立即知道自己的位置
```

#### Level 2：我要做什么？（行动）
```
→ 流程显示"启动投递"是下一步
→ 用户明确知道行动方向
```

#### Level 3：结果是什么？（预期）
```
→ 流程显示完整5步
→ 用户预期"启动后会自动投递"
```

## 💡 关键设计决策

### 1. 流程优先于按钮

**原因**：
- 用户需要理解系统如何工作
- 流程提供完整的操作地图
- 按钮只是执行工具，不是目的

### 2. 精简而非浮夸

**原因**：
- 过大的按钮会让人觉得"幼稚"
- 精简设计更专业
- 减少视觉疲劳

### 3. 折叠次要信息

**原因**：
- 降低首屏信息密度
- 让用户专注核心流程
- 保留完整功能，按需访问
