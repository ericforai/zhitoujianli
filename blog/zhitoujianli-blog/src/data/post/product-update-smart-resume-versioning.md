## 为什么需要简历版本管理？

在求职过程中，很多求职者会遇到这样的问题：

- **岗位要求不同**：产品经理岗位强调产品思维和数据分析，而市场经理岗位更看重营销经验和品牌建设
- **技能侧重点不同**：前端开发岗位需要React、Vue，后端开发岗位需要Java、Spring Boot
- **行业背景不同**：互联网行业看重快速迭代，传统行业更看重稳定性和流程规范

**传统做法的问题：**
- 使用同一份简历投递所有岗位，匹配度低
- 手动修改简历，效率低，容易出错
- 版本混乱，不知道哪个版本投递了哪个岗位

**解决方案：智能简历版本管理**

智投简历新增智能简历版本管理功能，让你可以为不同岗位创建多个简历版本，突出不同技能和经历，一键切换，让简历更精准匹配目标岗位。

## 新功能详解

### 1. 多版本简历创建

**功能说明：**

- 支持基于原始简历创建多个版本
- 每个版本可以有不同的侧重点和内容
- 版本命名：支持自定义版本名称（如"产品经理版"、"市场经理版"）

**使用场景：**

- **跨岗位投递**：同时投递产品经理和市场经理岗位
- **跨行业投递**：同时投递互联网和传统行业
- **技能突出**：为不同技能组合创建不同版本

**创建方法：**

1. 进入"简历管理"页面
2. 点击"创建新版本"
3. 选择基础版本（原始简历或已有版本）
4. 编辑内容，突出目标岗位所需技能和经历
5. 保存并命名版本

### 2. 智能内容适配

**功能说明：**

- AI自动分析目标岗位JD，识别关键技能和经历要求
- 基于分析结果，智能调整简历内容优先级
- 突出匹配度高的技能和经历，弱化不相关的经历

**适配逻辑：**

- **技能匹配**：自动识别JD中的技能要求，在简历中突出相关技能
- **经历匹配**：分析工作经历与岗位要求的匹配度，调整描述重点
- **关键词优化**：自动添加JD中的关键词，提升ATS通过率

**使用场景：**

- **快速适配**：上传JD，AI自动生成适配版本
- **精准匹配**：针对特定岗位，生成高度匹配的简历版本

### 3. 一键切换版本

**功能说明：**

- 在投递前，快速切换简历版本
- 支持预览不同版本的效果
- 自动记录每个版本的使用情况

**切换方法：**

1. 进入"投递管理"页面
2. 选择目标岗位
3. 点击"切换简历版本"
4. 选择匹配的版本
5. 确认并开始投递

**版本推荐：**

- AI会根据岗位JD，推荐最匹配的简历版本
- 显示匹配度评分，帮助选择最佳版本

### 4. 版本对比与管理

**功能说明：**

- 支持对比不同版本的差异
- 查看每个版本的投递记录和效果
- 支持版本删除、重命名、复制等操作

**对比功能：**

- **内容对比**：高亮显示不同版本的差异
- **效果对比**：对比不同版本的投递成功率、回复率
- **优化建议**：基于效果数据，提供版本优化建议

**管理功能：**

- **版本列表**：清晰展示所有版本
- **版本状态**：显示每个版本的使用情况
- **批量操作**：支持批量删除、导出等操作

## 如何使用？（快速上手）

### 步骤1：创建第一个版本

1. 上传原始简历
2. 进入"简历管理"页面
3. 点击"创建新版本"
4. 编辑内容，突出目标岗位所需技能
5. 保存并命名（如"产品经理版"）

### 步骤2：创建更多版本

1. 基于原始简历或已有版本，创建新版本
2. 针对不同岗位，调整内容重点
3. 保存并命名（如"市场经理版"、"前端开发版"）

### 步骤3：投递时切换版本

1. 进入"投递管理"页面
2. 选择目标岗位
3. 点击"切换简历版本"
4. 选择匹配的版本（或使用AI推荐）
5. 确认并开始投递

### 步骤4：跟踪效果

1. 在"投递记录"中查看不同版本的效果
2. 对比不同版本的投递成功率、回复率
3. 根据效果数据，优化版本内容

## 最佳实践与建议

### 1. 版本数量控制

- **推荐数量**：3-5个版本（根据投递岗位类型）
- **过多版本**：版本过多会增加管理成本，建议合并相似版本
- **过少版本**：如果只投递一种岗位，1-2个版本即可

### 2. 版本命名规范

- **清晰命名**：使用岗位名称或技能组合命名（如"产品经理版"、"React前端版"）
- **避免混淆**：不要使用"版本1"、"版本2"等模糊命名
- **及时更新**：如果版本内容有重大调整，及时更新命名

### 3. 内容差异化

- **技能突出**：不同版本突出不同的核心技能
- **经历调整**：根据岗位要求，调整工作经历描述重点
- **关键词优化**：每个版本包含目标岗位的关键词

### 4. 效果跟踪与优化

- **定期查看**：每周查看不同版本的投递效果
- **数据分析**：对比不同版本的回复率、面试邀请率
- **持续优化**：根据效果数据，调整版本内容

## 常见问题

### Q1: 可以创建多少个版本？

**A:** 理论上没有限制，但建议控制在3-5个版本，便于管理。

### Q2: 版本之间可以共享内容吗？

**A:** 可以。创建新版本时，可以选择基于原始简历或已有版本，共享基础内容。

### Q3: 如何知道哪个版本效果最好？

**A:** 在"投递记录"中，可以查看每个版本的投递成功率、回复率等数据，对比分析。

### Q4: 可以删除版本吗？

**A:** 可以。在"简历管理"页面，可以删除不需要的版本。注意：删除前请确认该版本没有正在进行的投递。

### Q5: 版本切换会影响已投递的记录吗？

**A:** 不会。已投递的记录会保留原版本的简历内容，不会因为版本切换而改变。
