## 能做什么
- 结构体检：模块完整性、顺序合理性、可读性与ATS友好性提示
- 关键词覆盖：与目标JD比对缺失与冗余关键词清单
- 量化改写：职责句式改为“动作-对象-方法-指标”的可验证表达
- 重复与冗余：去重建议与合并提示
- 风险雷区：敏感信息、涉密内容、时间空窗表达建议

## 使用流程
1) 上传现有简历（PDF/Word/纯文档）
2) 选择目标岗位或粘贴JD → 自动生成对齐清单
3) 一键生成“优化版简历草稿”与“打招呼语V2”草稿
4) 手动微调并保存 → 可导出PDF或继续投递

## 优化建议（示例）
- 将“负责XX”替换为“通过YY方法完成XX，结果ZZ（时长/比例/规模）”
- 保留3-5个最能体现岗位匹配度的项目，其余入“补充信息”
- 技能清单按“核心/熟练/了解”分级，避免堆砌

## 常见问题
- Q：会不会把我简历改得面试无法支撑？
  A：请确保所有表述均有事实依据；系统仅给出句式与结构建议，不建议虚构经历。
- Q：上传的隐私数据如何处理？
  A：请先脱敏再上传；我们建议在导入前移除身份证、家庭住址等与求职无关信息。

## 这类用户最需要上传诊断
- 不确定简历质量：不知道HR会怎么看这份简历
- 面试命中率低：明明有经历但描述不“打点”
- 关键词缺失：岗位要求说到了，但简历里没体现或不明显
- 转型/跨城：需重新组织叙事与可迁移能力

## 诊断结果包含哪些维度
### 1）结构完整性与可读性
- 模块齐全度（信息/教育/经历/项目/技能/附加）
- 顺序合理性（是否符合应届/社招/转型的阅读习惯）
- 文风统一性（第一/第三人称、标点/缩写一致）

### 2）ATS友好性与格式风险
- 是否存在图片化文字、复杂分栏、花哨排版
- 是否包含难被机器解析的元素（形状、图表）
- 建议改为文本化要点列举

### 3）关键词覆盖与岗位匹配
- 基于目标JD抽取关键词：职责词、技能词、场景词
- 覆盖度提示：缺失/冗余/弱覆盖
- 提示放置位置：技能清单/项目经历/标题

### 4）量化表达与成果呈现
- 将“负责XX”改写为“在[场景]通过[方法]完成[动作]，达成[指标]”
- 指标建议：时长/比例/性能/规模/业务结果/复盘沉淀

### 5）风险与合规
- 敏感信息（身份证、家庭住址、过度隐私）
- 涉密内容（商业机密、未披露数据）
- 夸大/虚构警示（避免不可证伪描述）

## 一份示例“诊断报告”结构（节选）
```
概览：整体评分（结构/ATS/关键词/量化/合规）
建议优先级：P0（必须修正）/ P1（强建议）/ P2（优化项）

P0 修正
- 将“教育经历”移至“技能”之后（社招优先展示产出）
- 删除身份证与详细住址等敏感信息

P1 强建议
- 项目A：增加目标/行动/结果，将“负责”改为“通过…将…从…到…”
- 技能清单分级：核心/熟练/了解，保留与JD高度相关项

P2 优化项
- 增加“复盘与沉淀”，便于展示可迁移能力
```

## FAQ（扩展）
- Q：可以给出统一分数吗？
  A：目前以“维度级评分+优先级清单”为主，更便于逐项修复。
- Q：关键词会不会让简历显得“刻意”？
  A：只保留与真实经历匹配的词；将关键词嵌入“行动/产出/指标”中更自然。
- Q：上传隐私如何保护？
  A：建议先脱敏再上传；我们会在诊断环节提示可能的风险点。
