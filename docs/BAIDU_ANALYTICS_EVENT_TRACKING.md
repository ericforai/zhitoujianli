# 百度统计事件跟踪 - 找不到utm_params事件？

## 🔍 问题说明

百度统计的"事件跟踪"功能可能：
1. 不在左侧菜单的明显位置
2. 需要先配置才能使用
3. 或者界面结构不同

## ✅ 解决方案：使用更简单的方法

### 方法1：查看"外部链接"详情（最推荐）⭐

这是**最简单、最直接**的方法：

#### 步骤1：进入"全部来源"页面
1. 登录百度统计
2. 左侧菜单：**来源分析** → **全部来源**

#### 步骤2：点击"外部链接"
在数据表格中，找到 **"来源类型"** 列，点击 **"外部链接"** 这一行

#### 步骤3：查看详细URL
点击后，会看到完整的URL列表，UTM参数就在URL中：
```
https://zhitoujianli.com/register?utm_source=wechat&utm_medium=article&utm_campaign=launch_2025&utm_content=launch_article
```

**优点：**
- ✅ 无需任何配置
- ✅ 数据完整准确
- ✅ 可以直接看到UTM参数

---

### 方法2：使用浏览器控制台检查（实时调试）

如果百度统计的事件跟踪找不到，可以用浏览器控制台直接检查：

#### 步骤1：访问测试链接
```
https://zhitoujianli.com/register?utm_source=wechat&utm_medium=article&utm_campaign=launch_2025&utm_content=launch_article
```

#### 步骤2：打开控制台（F12）
切换到"Console（控制台）"标签

#### 步骤3：查看日志
应该能看到：
```
📊 [UTM跟踪] 捕获到UTM参数: {utm_source: "wechat", utm_medium: "article", ...}
✅ 百度统计UTM参数已发送: {utm_source: "wechat", ...}
📊 百度统计UTM事件已发送: utm_source = wechat
📊 百度统计UTM事件已发送: utm_medium = article
```

如果看到这些日志，说明**代码正常工作**，UTM参数已经发送到百度统计。

#### 步骤4：手动检查百度统计对象
在控制台输入：
```javascript
// 检查百度统计是否加载
console.log('百度统计已加载:', typeof _hmt !== 'undefined');

// 检查UTM参数是否存储
console.log('UTM参数:', localStorage.getItem('zhitoujianli_utm_params'));
```
