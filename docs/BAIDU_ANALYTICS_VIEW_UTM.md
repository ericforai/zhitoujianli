# 百度统计查看UTM参数 - 详细步骤

## 🔍 问题：在"全部来源"页面看不到UTM参数

这是正常的！**"全部来源"页面只显示来源类型汇总**，需要点击进入详细页面才能看到UTM参数。

---

## ✅ 方法1：查看外部链接的详细URL（推荐）

### 步骤1：进入"全部来源"页面
1. 登录百度统计
2. 左侧菜单：**来源分析** → **全部来源**
3. 可以看到来源类型汇总（直接访问、外部链接等）

### 步骤2：点击"外部链接"
在数据表格中，找到 **"来源类型"** 列，点击 **"外部链接"** 这一行

### 步骤3：查看详细URL列表
点击后，会进入"外部链接"的详细页面，可以看到：
- 完整的URL列表
- 每个URL的访问次数
- **UTM参数会显示在URL中**

**示例：**
```
https://zhitoujianli.com/register?utm_source=wechat&utm_medium=article&utm_campaign=launch_2025&utm_content=launch_article
```

### 步骤4：筛选和分析
- 可以按URL筛选
- 可以看到每个带UTM参数的URL的访问次数
- 可以对比不同UTM参数的效果

---

## ✅ 方法2：查看"搜索词"（如果适用）

如果UTM参数是通过搜索或特定方式传递的，也可能出现在"搜索词"中：

1. 在"全部来源"页面
2. 点击 **"搜索词"** 标签
3. 查看是否有包含UTM参数的记录

---

## ✅ 方法3：查看事件跟踪（实时可见）

### 步骤1：进入事件跟踪页面
1. 登录百度统计
2. 左侧菜单：**事件跟踪**

### 步骤2：查找UTM事件
在事件列表中查找：
- **事件类别**：`utm_params`
- **事件动作**：`utm_source`, `utm_medium`, `utm_campaign`, `utm_content`
- **事件标签**：对应的值（如 `wechat`, `article`）

### 步骤3：查看数据
- 可以看到每个UTM参数的触发次数
- 可以看到不同UTM参数值的分布

---

## 🔧 调试方法：检查代码是否正常工作

### 步骤1：打开浏览器控制台
1. 访问带UTM参数的链接：
```
https://zhitoujianli.com/register?utm_source=wechat&utm_medium=article&utm_campaign=launch_2025&utm_content=launch_article
```

2. 按 `F12` 打开开发者工具
3. 切换到 **"Console"（控制台）** 标签

### 步骤2：查看日志输出
应该能看到以下日志：

```
[UTMTracker] 捕获到UTM参数: {utm_source: "wechat", utm_medium: "article", ...}
✅ 百度统计转化事件已发送: sign_up
```

### 步骤3：检查百度统计代码
在控制台输入以下命令，检查百度统计是否加载：

```javascript
// 检查百度统计是否加载
console.log('百度统计已加载:', typeof _hmt !== 'undefined');

// 检查UTM参数是否发送
console.log('UTM参数:', localStorage.getItem('zhitoujianli_utm_params'));
```

### 步骤4：手动触发测试
在控制台输入以下命令，手动发送UTM参数到百度统计：

```javascript
// 手动发送UTM参数到百度统计
if (typeof _hmt !== 'undefined') {
  _hmt.push(['_trackEvent', 'utm_params', 'utm_source', 'wechat']);
  _hmt.push(['_trackEvent', 'utm_params', 'utm_medium', 'article']);
  _hmt.push(['_trackEvent', 'utm_params', 'utm_campaign', 'launch_2025']);
  console.log('✅ UTM参数已手动发送到百度统计');
} else {
  console.error('❌ 百度统计未加载');
}
```

## ⚠️ 常见问题

### Q1: 为什么"外部链接"只有1个浏览量？
**A:** 这说明只有1次访问来自外部链接。如果访问量很少，数据可能不明显。建议：
- 多访问几次测试链接
- 等待更多真实访问
- 检查UTM链接是否正确使用

### Q2: 点击"外部链接"后看不到详细URL？
**A:** 可能的原因：
- 访问量太少，数据还未更新（百度统计有延迟）
- 浏览器缓存问题，清除缓存后重试
- UTM参数没有正确传递

### Q3: 事件跟踪中看不到 `utm_params` 事件？
**A:** 检查：
1. 代码是否已部署（清除浏览器缓存）
2. 控制台是否有错误
3. 百度统计代码是否正常加载
