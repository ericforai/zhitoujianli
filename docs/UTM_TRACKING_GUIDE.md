# UTM跟踪链接使用指南

## 📋 公众号启动文章跟踪链接

### 注册页面跟踪链接

**推荐使用的完整链接：**

```
https://zhitoujianli.com/register?utm_source=wechat&utm_medium=article&utm_campaign=launch_2025&utm_content=launch_article
```

### UTM参数说明

| 参数           | 值               | 说明               |
| -------------- | ---------------- | ------------------ |
| `utm_source`   | `wechat`         | 来源：微信公众号   |
| `utm_medium`   | `article`        | 媒介：文章         |
| `utm_campaign` | `launch_2025`    | 活动：2025启动活动 |
| `utm_content`  | `launch_article` | 内容标识：启动文章 |

## 📊 如何查看数据

### Google Analytics 4（推荐）

#### 1. 实时报告

- 登录 [Google Analytics](https://analytics.google.com/)
- **报告** → **实时**
- 查看来源/媒介：`wechat / article`

#### 2. 事件报告

- **报告** → **参与度** → **事件**
- 查找事件：`sign_up`
- 查看注册转化次数和UTM参数

#### 3. 自定义UTM报告

- **探索** → 创建新报告
- 添加维度：`utm_source`, `utm_medium`, `utm_campaign`, `utm_content`
- 添加指标：`事件数`, `用户数`, `会话数`

### 百度统计（三种查看方式）

#### ⚠️ 重要：百度统计需要特殊设置

百度统计查看UTM参数有**三种方式**，推荐使用**方法1**（最简单）：

#### 方法1：URL参数（推荐，无需配置）✅

**原理：** 百度统计会自动识别URL中的UTM参数

**查看步骤：**

1. 登录 [百度统计](https://tongji.baidu.com/)
2. 左侧菜单：**来源分析** → **全部来源**
3. 点击 **"搜索词"** 或 **"外部链接"**
4. 可以看到完整的URL，包括UTM参数

**优点：**

- ✅ 无需任何后台配置
- ✅ 自动识别URL参数
- ✅ 数据完整准确

---

#### 方法2：事件跟踪（可选，需要找到正确位置）⚙️

**原理：** 使用百度统计的事件跟踪功能

**查看步骤：**

1. 登录百度统计
2. 左侧菜单查找以下位置之一：
   - **事件分析** → **自定义事件**（推荐）
   - **行为分析** → **事件跟踪**
   - **管理** → **事件跟踪设置**
3. 在自定义事件页面查找：
   - 事件类别：`utm_params`
   - 事件动作：`utm_source`, `utm_medium` 等
   - 事件标签：对应的值（如 `wechat`, `article`）

**注意：**

- ⚠️ 如果找不到"事件跟踪"功能，这是正常的
- ⚠️ 某些版本的百度统计可能没有独立的事件跟踪报告
- ✅ **推荐使用方法1（外部链接详情），更简单直接！**

**优点：**

- ✅ 无需后台配置
- ✅ 可以跟踪多个UTM参数
- ✅ 数据实时可见

---

#### 方法3：自定义变量（需要后台配置）⚙️

**原理：** 使用百度统计的自定义变量功能

**配置步骤：**

1. 登录百度统计
2. 左侧菜单：**管理** → **自定义变量**
3. 创建5个自定义变量（访问级）：

   | 变量序号 | 变量名称     | 变量类型 | 说明      |
   | -------- | ------------ | -------- | --------- |
   | 1        | utm_source   | 访问级   | UTM来源   |
   | 2        | utm_medium   | 访问级   | UTM媒介   |
   | 3        | utm_campaign | 访问级   | UTM活动   |
   | 4        | utm_term     | 访问级   | UTM关键词 |
   | 5        | utm_content  | 访问级   | UTM内容   |

4. 等待24小时后查看数据

**查看步骤：**

1. 左侧菜单：**自定义变量**
2. 可以看到每个变量的值和访问次数

**优点：**

- ✅ 可以按UTM参数筛选和分组
- ✅ 数据展示更清晰

**缺点：**

- ❌ 需要后台配置
- ❌ 配置后需要等待24小时

## 📈 关键指标

### 访问指标

- **会话数**：来自公众号的访问次数
- **用户数**：来自公众号的独立用户数
- **页面浏览量**：注册页面的访问次数

### 转化指标

- **注册转化数**：完成注册的用户数
- **注册转化率**：注册转化数 / 访问注册页面数

## 📊 如何查看UTM参数的具体数据

### 在百度统计中查看

1. **来源分析** → **全部来源** → 点击 **"外部链接"**
2. 查看详细URL列表，UTM参数就在URL中
3. 可以看到每个URL的：
   - 浏览量(PV)
   - 访客数(UV)
   - 跳出率
   - 平均访问时长

### 在Google Analytics中查看

#### 重要提示

如果找不到 `sign_up` 事件，这是正常的！可能的原因：

- ⏰ 还没有人注册，所以没有事件数据
- ⏰ 数据延迟：GA4有24-48小时的数据延迟
- ✅ **先完成一次注册测试，触发事件！**

#### 查看步骤

**方法1：实时报告（立即查看）**

1. **报告** → **实时概览**
2. 向下滚动，查找"事件"部分
3. 应该能看到 `sign_up` 事件（如果刚刚注册）

**方法2：事件报告（查看历史数据）**

1. **生命周期** → **互动度** → **事件**
2. 在事件列表中查找 `sign_up`
3. 点击事件名称，查看详细数据和UTM参数

**方法3：创建自定义UTM报告**

1. **探索** → 创建新报告
2. 添加维度：`UTM来源`, `UTM媒介`, `UTM活动`, `UTM内容`
3. 添加指标：`事件数`, `用户数`, `转化率`
4. 筛选：`事件名称` = `sign_up`

**详细的事件调试指南，请查看：**

- [Google Analytics 4 事件调试指南](./GA4_EVENT_DEBUG.md)

### 数据分析技巧

- **对比不同来源**：`utm_source=wechat` vs `utm_source=zhihu`
- **对比不同内容**：`utm_content=launch_article` vs `utm_content=menu_register`
- **计算转化率**：注册转化数 / 访问注册页面数
